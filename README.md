# Loona Bridge — аддон для Home Assistant

Docker-аддон запускает **Node + Puppeteer + `bridge.js`**, подключается по WebSocket к Core. Образ на **Debian bookworm-slim**: ставится **Chromium** и при возможности скачивается **Google Chrome** (.deb arm64/amd64) — без Chrome/Chromium с кодеками Loona часто даёт «no decoded video» и `framesDec=0`. **`run.sh`** сам выбирает `google-chrome-stable`, если он установился при сборке.

Опционально **`chrome_path`** в настройках аддона, если браузер нестандартный.

Состав репозитория:

- `repository.yaml` — метаданные для Supervisor.
- `loona_bridge/` — сам аддон (`config.yaml`, `Dockerfile`, `run.sh`, каталог `bridge/` — копия из интеграции).

## Обновление кода бриджа

После правок в [`../bridge/`](../bridge/) скопируйте файлы в образ аддона перед сборкой:

```bash
cp -R /path/to/custom_components/loona/bridge ./loona_bridge/
```

## Способ 1: Git-репозиторий (удобно для HAOS)

1. Выложите каталог **`addon_repository`** (этот репозиторий) на GitHub/GitLab (корень репозитория должен содержать `repository.yaml` и папку `loona_bridge`).
2. В Home Assistant: **Настройки** → **Дополнения** → **Магазин дополнений** → меню **⋮** → **Репозитории**.
3. Добавьте URL вида `https://github.com/<user>/<repo>` и сохраните.
4. Обновите страницу магазина — появится аддон **Loona Agora Bridge**.
5. Установите аддон. Первая сборка на Raspberry Pi может занять много времени.

**Если в логе сборки видно только `apt-get` и ошибку 127:** Supervisor всё ещё берёт **старый Dockerfile** из Git. Откройте в браузере сырой файл `loona_bridge/Dockerfile` на GitHub — первые строки должны быть `apk add`, не `apt-get`. Затем push на `main`, в HA: репозиторий → обновить, аддон переустановить (версия **0.1.3+**).

**`npm: not found` при сборке:** Supervisor подставляет базовый образ без Node — в образе ставятся **`nodejs` и `npm` из Alpine через `apk`** (с версии аддона **0.1.3**).
6. Запуск: **Выключено вручную** по умолчанию (`manual_only`) — перед просмотром камеры включите аддон (**Старт**).

## Способ 2: Локальная папка `/addons` (HAOS + Samba)

1. Установите аддон **Samba share** (или откройте SSH) и получите доступ к общей папке **`addons`**.
2. Скопируйте всю папку **`loona_bridge`** в `/addons/loona_bridge` на хосте (рядом появится аддон без отдельного URL репозитория — см. документацию HA по локальным дополнениям).
3. **Проверка**: в каталоге должны быть `config.yaml`, `Dockerfile`, `build.yaml`, `run.sh`, `bridge/`.
4. В магазине дополнений нажмите **Проверить наличие обновлений**, затем установите и запустите аддон.

*(Точное имя шары и путь зависят от версии HA; при сомнениях используйте способ 1.)*

## Интеграция Loona в Core

1. **Настройки** → **Устройства и службы** → **Loona** → **Настроить**.
2. Включите **«Видео-бридж вне Core»** / **Run video bridge outside Core** — иначе Core снова запустит свой `bridge.js` и в канале Agora окажутся два клиента.
3. Сохраните. При необходимости перезагрузите интеграцию или HA.

## Опции аддона

| Параметр          | Смысл |
|-------------------|--------|
| `chrome_path`     | Пусто — авто (`chromium` / `google-chrome` в образе). Укажите полный путь к браузеру с H.264, если ставили свой пакет. |
| `fps`             | Частота JPEG (по умолчанию 10 — мягче для Pi с 2 ГБ RAM). |
| `jpeg_quality`    | Качество JPEG 0.35–0.95. |

Переменная **`HA_WS_HOST`** (не в UI): hostname Core для WebSocket. По умолчанию `homeassistant` — стандартная DNS-запись в сети Supervisor.

## Типовой порядок действий

1. Включить выключатель камеры Loona в HA.
2. Запустить аддон **Loona Agora Bridge** (если он на `manual_only`).
3. Дождаться превью камеры; при таймауте смотреть логи аддона и Core (`bridge_external_addon` должен быть включён).

## Ограничения

- Образ ставит **Chromium из Debian** — на части сборок нет нужного **H.264** для WebRTC; тогда нужен другой бинарь (Chrome/Brave для arm64, если доступен) и поле **`chrome_path`**.
- RAM: на Pi с **2 ГБ** возможны OOM при одновременной нагрузке на HA — уменьшайте `fps` и число тяжёлых дополнений.
